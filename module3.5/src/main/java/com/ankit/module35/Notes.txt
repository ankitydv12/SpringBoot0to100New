Commit-->3.5.2 Class Based Projection

      * Class-based Projections (DTOs/Records):
        You use a Data Transfer Object (DTO) or a Java record with a constructor whose parameter names match the property names of the entity fields you want to retrieve.
        In this case, Spring uses reflection to instantiate and populate the DTOs directly, so no proxying happens.
      * Dynamic Projections:
        This allows you to define the projection type (interface or class) at runtime by passing the Class<T> as a parameter to the repository method.


                             * Class-based Projections (DTOs/Records) *

@Repository
