springdoc-openapi java library helps to automate the generation of API documentation using spring boot projects.
springdoc-openapi works by examining an application at runtime to infer API semantics based on spring configurations, class structure and various annotations.

   <dependency>
      <groupId>org.springdoc</groupId>
      <artifactId>springdoc-openapi-starter-webmvc-ui</artifactId>
      <version>2.8.17</version>
   </dependency>

The Swagger UI page will then be available at http://server:port/context-path/swagger-ui.html
the OpenAPI description will be available at the following url for json format: http://server:port/context-path/v3/api-docs

Very Important:
When using a Global Response Handler (ResponseBodyAdvice), you must restrict its scope using @RestControllerAdvice(basePackages = "com.ankit.module4").
    Problem: Without a defined scope, the handler intercepts internal endpoints like Swagger and Actuator.
    Swagger/Actuator endpoints return raw byte arrays (byte[]). If the Global Handler wraps these bytes into a custom ApiResponse object,
    the internal 'ByteArrayHttpMessageConverter' fails to cast the ApiResponse back to byte[], causing a ClassCastException.
    Solution: Restricting the package ensures the handler only processes your application's controllers.

 Fix - Scope your Advice (Recommended)
   @RestControllerAdvice(basePackages = "com.ankit.module4")
✅       Why it works:  This is the cleanest fix. It tells Spring: only intercept controllers inside MY package.
        Swagger and Actuator live in different packages, so they are automatically excluded.

Some Properties :
springdoc.api-docs.path= /apidoc  # Changes the default JSON documentation path from /v3/api-docs to /apidoc

springdoc.show-actuator=true     # Includes Spring Boot Actuator endpoints (like /health, /info) in the Swagger UI documentation

springdoc.swagger-ui.path= /swagger  # Changes the default Swagger UI HTML access path from /swagger-ui.html to /swagger

springdoc.swagger-ui.try-it-out-enabled=true  # Automatically expands the "Try it out" section for all API operations by default
