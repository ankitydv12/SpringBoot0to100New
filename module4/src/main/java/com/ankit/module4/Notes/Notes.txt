RESTCLIENT

1. What is RestClient?
RestClient is Spring's synchronous HTTP client used to call another REST API from a Spring Boot application.
It is mainly used when one service needs to communicate with another service.

Examples:
- Employee service calling another microservice
- One backend application fetching data from an external API

RestClient is a modern alternative to RestTemplate for synchronous communication.


2. Why do we use RestClient?
- To consume external REST APIs
- To call another microservice from our application
- To send HTTP requests like GET, POST, PUT, PATCH, DELETE
- To map JSON response directly into DTO classes


3. Basic flow of RestClient implementation
Step 1: Add base URL in application.properties
Example:
employeeService.base.url=http://localhost:8081/

Step 2: Create RestClient configuration class

Step 3: Expose RestClient as a Spring Bean

Step 4: Inject RestClient inside a service/client class

Step 5: Use get(), post(), put(), patch(), delete() methods to call APIs

Step 6: Convert response JSON into DTO / ApiResponse<T>


4. RestClientConfig explained
In this project:
RestClientConfig creates and configures the RestClient bean.

Code idea:
@Configuration
public class RestClientConfig {

    @Value("${employeeService.base.url}")
    private String baseUrl;

    @Bean
    @Qualifier("getEmployeeServiceRestClient")
    RestClient getEmployeeServiceRestClient() {
        return RestClient.builder()
                .baseUrl(baseUrl)
                .defaultHeader(HttpHeaders.CONTENT_TYPE, APPLICATION_JSON_VALUE)
                .build();
    }
}

Explanation:
- @Configuration
  Tells Spring that this class contains bean definitions.

- @Value("${employeeService.base.url}")
  Reads the base URL from properties file.
  This avoids hardcoding URL inside service logic.

- @Bean
  Creates a RestClient object and registers it in Spring container.

- @Qualifier("getEmployeeServiceRestClient")
  Gives a name to this bean.
  Useful when multiple RestClient beans exist.

- .baseUrl(baseUrl)
  Sets the common URL for all requests.
  After this, we can write .uri("employees") instead of full URL.

- .defaultHeader(HttpHeaders.CONTENT_TYPE, APPLICATION_JSON_VALUE)
  Adds default Content-Type as application/json.
  Helpful for requests sending JSON body.




5. How RestClient works in this project
Project structure:
- RestClientConfig creates the bean
- EmployeeClientImp uses the bean
- RestClientController calls EmployeeClientImp

Flow:
Controller -> Client/Service -> RestClient -> External API


6. GET method with RestClient
GET is used to fetch/read data.

Example for all employees:
ApiResponse<List<EmployeeDTO>> response = restClient.get()
        .uri("employees")
        .retrieve()
        .body(new ParameterizedTypeReference<>() {});

return response.getData();

Explanation:
- restClient.get() -> selects HTTP GET method
- .uri("employees") -> endpoint path
- .retrieve() -> sends request and gets response
- .body(...) -> converts JSON response to Java object
- response.getData() -> extracts actual data from ApiResponse wrapper

Example for one employee by id:
ApiResponse<EmployeeDTO> response = restClient.get()
        .uri("employees/{id}", id)
        .retrieve()
        .body(new ParameterizedTypeReference<>() {});

return response.getData();


7. POST method with RestClient
POST is used to create new data.

Example:
ApiResponse<EmployeeDTO> response = restClient.post()
        .uri("employees")
        .body(empRequest)
        .retrieve()
        .body(new ParameterizedTypeReference<>() {});

return response.getData();

Explanation:
- post() -> selects HTTP POST method
- body(empRequest) -> sends EmployeeDTO object as JSON
- retrieve() -> executes request
- body(...) -> converts response back into ApiResponse<EmployeeDTO>

restClient
    .method()
    .uri(...)
    .body(...)      // only for POST, PUT, PATCH
    .retrieve()
    .body(...);



12. Why ParameterizedTypeReference is used?
When response contains generic types like:
- ApiResponse<EmployeeDTO>
- ApiResponse<List<EmployeeDTO>>

Java type erasure makes generic type mapping difficult at runtime.
ParameterizedTypeReference helps Spring understand the exact response type.

Example:
.body(new ParameterizedTypeReference<ApiResponse<List<EmployeeDTO>>>() {})


13. Advantages of RestClient
- Clean and readable API
- Easy bean configuration
- Good for synchronous communication
- Better modern approach in Spring
- Easily integrates with DTO mapping
- Useful for microservice architecture


14. Short interview answer
RestClient is Spring's synchronous HTTP client used to consume REST APIs from a Spring Boot application.
It is configured using RestClient.builder(), usually in a configuration class. We can set base URL, headers, and create a reusable bean.
Then we inject that bean into a service class and use methods like get(), post(), put(), patch(), and delete() to call external APIs and map the response into DTOs.

15. One-line summary
RestClient is used to call external REST APIs from a Spring Boot application in a clean, reusable, and synchronous way.
