# Spring Security Notes

## Customizing SecurityFilterChain (WebSecurityConfig)
In Spring Security, the `SecurityFilterChain` is the core of web security configuration.
We can customize it by defining a `@Bean` of type `SecurityFilterChain` in a configuration class.

Key components from the configuration:
- `@Configuration` & `@EnableWebSecurity`: These annotations mark the class as a security configuration class and enable Spring Security's web security support.
- `SecurityFilterChain` Bean: By defining this bean, we can override the default Spring Boot auto-configuration for security and provide our own custom rules.
- `HttpSecurity`: The `HttpSecurity` builder is used to configure web based security for specific http requests.
- `httpSecurity.formLogin(Customizer.withDefaults())`: This specific line enables the default form-based authentication (the default Spring Security login page).
- `httpSecurity.build()`: Finally, we call `.build()` to create and return the customized `SecurityFilterChain` instance.

    @Bean
    SecurityFilterChain securityFilterChain(HttpSecurity httpSecurity) throws Exception
    {
        httpSecurity.formLogin(Customizer.withDefaults());
        return  httpSecurity.build();
    }
